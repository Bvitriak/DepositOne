# DepositOne

**Информационная система учета депозитов физических лиц**
Современная веб-система для автоматизации банковских операций по вкладам. Предназначена для небольших банков, кредитных союзов и финансовых организаций.
Система полностью охватывает жизненный цикл депозита - от регистрации вкладчика до начисления процентов, оформления договора и формирования аналитической отчётности.

---

## Основные возможности

- Полноценное управление вкладчиками
- Открытие, редактирование и пополнение вкладов
- Автоматическое формирование номеров договоров
- Начисление процентов с капитализацией
- Учёт кассовых операций
- Планирование возвратов
- Многоуровневая аналитика и cash flow
- JWT-авторизация + Flask-Login
- Отдельный JSON API для интеграций
- Адаптивный и современный интерфейс

## Описание основных компонентов
### Core Service

- `app.py` - создание и запуск Flask-приложения

- Формы создания и редактирования сущностей

## Подробная информация по каждому файлу
### Корневые файлы
- `app.py`: Главный файл приложения. Создаёт и настраивает Flask-приложение, подключает все маршруты, систему авторизации, обработку базы данных и запускает сервер.
- `init_db.py`: Скрипт для первоначальной инициализации базы данных. Создаёт все необходимые таблицы.
- `requirements.txt`: Список всех Python-библиотек, необходимых для работы проекта.

### Core Service
#### `config/`
- `about.json`: Содержит основную информацию о проекте: название, версию, описание, список модулей, функций и технологий. Используется на странице "О проекте".
- `config.py`: Загружает настройки из файла .env и формирует конфигурацию Flask.

#### `models/`
- `base.py`: Базовый класс для всех моделей. Содержит общие методы работы с базой данных.
- `depositor.py`: Модель для работы с таблицей вкладчиков: создание, обновление, удаление и получение списка вкладчиков.
- `deposit.py`: Модель для работы с таблицей вкладов: создание, обновление, удаление, пополнение и получение информации о вкладах.
- `contract.py`: Модель для работы с договорами: создание договора, проверка существования, поиск по номеру и получение полных данных договора с информацией о вкладчике и вкладе.
- `user.py`: Модель пользователя системы, интегрированная с Flask-Login.
- `dto.py`: Data Transfer Objects - чистые структуры данных, используемые для передачи информации между слоями приложения.

#### `routes/`
- `routes_register.py`: Централизованная регистрация всех маршрутов в приложении.
- `routes_public.py`: Обработчики публичных страниц, доступных без авторизации: главная страница и страница "О проекте".
- `routes_depositors.py`: Все маршруты, связанные с вкладчиками: список, создание, редактирование и удаление вкладчиков.
- `routes_deposits.py`: Маршруты для работы с вкладами: список, создание, редактирование, пополнение, начисление процентов и удаление.
- `routes_contracts.py`: Маршруты для работы с договорами: просмотр списка и создание договора по существующему вкладу.
- `routes_analytics.py`: Маршруты для дашборда, финансовых отчётов и плана возвратов.
- `routes_helpers.py`: Вспомогательные функции: валидация форм, обработка flash-сообщений, получение записей или 404 и т.д.

#### `services/`
- `depositor_service.py`: Бизнес-логика работы с вкладчиками: создание, обновление, удаление с проверкой уникальности паспортных данных и обработкой ошибок целостности БД.
- `deposit_service.py`: Бизнес-логика вкладов: создание, обновление, пополнение суммы вклада и запись транзакции.
- `contract_service.py`: Логика создания договоров: генерация уникального номера договора по дате, проверка, что договор ещё не создан для данного вклада.
- `interest_service.py`: Расчёт и начисление процентов по вкладу, учёт капитализации, создание записей о начислениях и транзакциях.
- `report_service.py`: Формирование финансовых отчётов и сводки по вкладчикам.

#### `utils/`
- `fallbacks.py`: Система обработки ошибок. Позволяет возвращать понятные сообщения пользователю при сбоях базы данных, бизнес-ошибках или пустых результатах.
- `types.py`: Определяет тип ServiceResult - стандартную структуру ответа всех сервисных функций.

#### `templates/`
- Все HTML-шаблоны интерфейса.

#### `static/`
- Статические файлы: style.css и main.js.

#### `queries/`
- `list_queries.py`: Модуль сложных поисковых и аналитических запросов к базе данных.

- `db.py`: Модуль работы с SQLite. Реализует master/slave архитектуру: все изменения идут в master, чтение — из slave-реплики. Поддерживает WAL-режим для лучшей производительности.
- `constants.py`: Хранит все константы проекта: статусы вкладов, валюты, типы операций, настройки округления и т.д.
- `api.py`: REST API эндпоинты, защищённые JWT. Предоставляет данные для дашборда, списков вкладчиков и вкладов внешним системам или фронтенду.
- `about_data.py`: Загружает данные из about.json и передаёт их в шаблон страницы "О проекте".
- `auth.py`: Всё, что связано с авторизацией: регистрация новых пользователей, вход/выход, проверка пароля, генерация и обновление JWT-токенов.
- `decorators.py`: Содержит пользовательские декораторы для Flask-маршрутов.

### Supporting Service
#### `services/`
- `analytics_service.py`: Сервис аналитики: формирует данные для дашборда, метрики, список возвратов на завтра и пагинированный список вкладов.
- `report_service.py`: Сервис отчётов: расчёт движения денежных средств за разные периоды.
- `auth_service.py`: Сервис авторизации через API
- `notification_service.py`: Сервис отправки уведомлений.

#### `utils/`
- `fallbacks.py`: Модуль обработки ошибок и формирования стандартных ответов для вспомогательного сервиса.
- `pagination.py`: Вспомогательный модуль для работы с пагинацией.

- `main.py`: Запускает FastAPI-приложение и регистрирует все эндпоинты вспомогательного сервиса.
- `schemas.py`: Pydantic-модели для валидации данных, приходящих в API.
- `db.py`: Простое подключение к базе данных для supporting-сервиса.
- `constants.py`: Константы, специфичные для вспомогательного сервиса.

## Используемые технологии

- **Backend**: Python 3.12, Flask, FastAPI
- **База данных**: SQLite 3
- **Авторизация**: Flask-Login + Flask-JWT-Extended
- **Шаблоны**: Jinja2
- **Frontend**: HTML5, CSS3, JavaScript
- **Валидация**: Pydantic

## Установка и запуск
1. Клонирование и подготовка
```commandline
git clone https://github.com/Bvitriak/DepositOne.git
cd deposit_bank
```
2. Установка зависимостей
```commandline
pip install -r requirements.txt
```
3. Настройка окружения
```commandline
cp .env.example .env
```
5. Инициализация базы данных
```commandline
python init_db.py
```
6. Запуск основного приложения
```commandline
python app.py
Приложение будет доступно по адресу: http://127.0.0.1:5000
```
